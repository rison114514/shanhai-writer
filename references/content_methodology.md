# 内容方法论

## 选题方法论

### HKR质检法
- H (Happy) 足够有趣有悬念？
- K (Knowledge) 有信息量？
- R (Resonance) 能戳中情绪？
S级三项兼备，及格至少两项。

### 角色代入法
审视选题时分别代入：一个很忙的普通用户、一个爱玩的朋友、一个焦虑的学习者。

## 文章原型

### 调查实验型
亲自下场做一件事，边做边报道。核心：过程叙事 + 层层发现 + 真实反应。

### 产品体验型
带着读者一起体验产品。核心：场景驱动 + 真实感受 + 自然对比。

### 现象解读型
观察现象层层深入。核心：好奇心驱动 → 研究 → 知识掉落 → 哲学升维。

### 工具分享型
用个人故事包裹工具推荐。核心：故事铺垫 → 自然引出 → 使用过程 → 效果展示。

## 内容节奏

每次围绕主线偏出去一点，让读者喘口气看案例，再用一句话拉回来。扣主线句要高频。最忌讳突然偏离很远再硬拽回来。

## 创意案例工作法

包装成微型故事：亮出挑战 → 展示脑洞 → 秀出过程 → 引爆结果。案例必须有独立的观赏价值和启发价值。
