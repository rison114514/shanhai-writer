# 山海写作 (Shanhai Writer)

> 一套完整的 AI 辅助写作工具包。从创作到降 AI 率，逐段协作，全流程覆盖。

基于对朱雀等 AI 检测器的实测研究——从 100% AIGC 到全人工特征，4 版迭代。

## 快速开始

### 创作模式
```
「写一篇关于 XX 的文章」
→ 自动进入逐段协作：你给思路，AI 填空，不生成完整文章
```

### 降 AI 模式
```
「帮我把这篇文章降到全人工」
→ 自动进入 5 步快速路径：砍结构 → 注噪声 → 改语调 → 碎叙事 → 环状回扣
```

## 技能架构

```
shanhai-writer
├── 创作模式 → rui-writer（逐段协作写作）
│   ├── 人给骨架，AI 填空，不扩写
│   ├── 10 条有效方法
│   └── 继承 zhang-writer（16 条公理 / 不修正原则 / 风格内核）
│
└── 降AI模式 → de-ai（5 步快速路径）
    ├── Step 1: 砍结构（删除 AI 过渡词 + 打散三段论）
    ├── Step 2: 注噪声（8 种有效噪声，3 种无效噪声黑名单）
    ├── Step 3: 改语调（降权威 + 自推翻）
    ├── Step 4: 碎叙事（打乱信息推进顺序）
    ├── Step 5: 环状回扣（开头无关细节在结尾响）
    └── 执行纪律 7 条（确保效果一致性）
```

## 核心方法

### 有效噪声（8 种）

| # | 类型 | 示例 |
|---|---|---|
| 1 | 生理细节 | 「宫保鸡丁。花生多，鸡丁少。」 |
| 2 | 口是心非 | 「拇指放在屏幕右边准备划走了。但是没划。」 |
| 3 | 具体失败+精确时间 | 「三十秒。之前搞了二十分钟。4点47到5点07。」 |
| 4 | 叙事故意打断 | 「中间回了条微信。同事问我下午的会还开不开。」 |
| 5 | 自打断吐槽 | 「等一下，咖啡凉了这件事我之前是不是说过。」 |
| 6 | 中途自我怀疑 | 「这是教程吗。网上已经有一百篇了。」 |
| 7 | 口是心非链 | 先说不拉扯→立刻承认拉扯 |
| 8 | 失败清单收束 | 结尾盘点所有失败，不总结观点 |

### 无效噪声（黑名单）

| 类型 | 示例 | 为什么无效 |
|---|---|---|
| 通用化生理细节 | 「泡了杯咖啡」 | AI 训练数据高频模式 |
| 通用化类比 | 「用了十年 Windows」 | 噪声模板 |
| 通用化自嘲 | 「我也在反思」 | AI 也会模仿 |

## 原理

朱雀等 AI 检测器不是判断「像不像人」，是判断「像不像某个 AI 训练集里的文本」。公众号 AI 科技评测是朱雀训练数据最密集的类型。

降 AI 率的核心不是「更像人」，是**让文章偏离朱雀见过的 AI 文本分布**。
